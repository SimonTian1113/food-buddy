# 🍽️ 食通天 FoodBuddy

> 餐厅防踩雷验证工具 —— 从网上种草到出发前验证

[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

**不是美食推荐，而是帮你验证「这家店到底值不值得去」。**

当你在小红书、抖音被某家店种草，心里却嘀咕"会不会踩雷"时，FoodBuddy 会跨平台采集真实信号，做交叉验证，去滤镜、识套路，最终给你一个明确的判断。

---

## ✨ 核心能力

### 跨平台交叉验证
自动搜索并对比多个平台的真实数据：
- **Google Maps** — 全球通用评分
- **OpenRice** — 香港本地食客真实评价（权重最高）
- **大众点评** — 内地游客视角
- **小红书** — 热度参考（营销属性强，谨慎采信）
- **TripAdvisor** — 国际游客评价

### 营销噪音过滤
7 层检测逻辑自动识别并过滤：
- SEO 农场站（知乎、马蜂窝、百家号等）
- 模板化种草文（"绝绝子"、"yyds"、"全网爆火"）
- 标题党、纯情绪无实质内容的评论

### 老饕探案组 — 6 位 AI 探员

我们不是普通推荐系统，是一支**老饕探案组**。组长是退休美食编辑，嘴刁、眼光毒、不信广告信证据。手下 6 位探员，各有绝活：

| 探员 | 花名 | 人设 | 职责 |
|------|------|------|------|
| 🔍 线索猎犬 | 闻锅 | 嗅觉极强的侦查员 | 跨平台采集证据，闻得出锅气也闻得出水分 |
| 🔥 拆台师 | 戳泡 | 嘴毒但证据更毒 | 专戳"看起来很厉害"的泡泡，找平台矛盾 |
| 📝 评论法医 | 验词 | 冷静挑剔的细节控 | 验评论的尸检报告，找不可伪造信号 |
| 🏠 街坊雷达 | 熟门 | 本地混久了的朋友 | 判断是本地复购盘还是游客打卡盘 |
| 🎭 场景裁剪师 | 挑局 | 会在冲动前拦朋友的人 | 判断这次去值不值，不是不能吃但不适合你 |
| ⚖️ 收口官 | 定盘 | 见过太多网红店还清醒的人 | 拍板定案：值得去 / 谨慎去 / 不建议 |

### 加权裁决算法
- OpenRice 权重 30%（本地最可信）
- Google Maps 权重 25%
- 大众点评权重 20%
- 小红书权重 15%
- TripAdvisor 权重 10%（游客偏差大）

营销惩罚：重度营销 -0.15，中度 -0.08

## 🚀 快速开始

### 安装

```bash
# 克隆仓库
git clone https://github.com/songqiaotian/food-buddy.git
cd food-buddy

# 安装依赖（可选，用于浏览器搜索模式）
pip install playwright requests beautifulsoup4
playwright install chromium
```

### 运行

```bash
python3 scripts/orchestrator.py
```

## 🔧 技术说明

### 搜索模式

**浏览器搜索（推荐）**：使用 Playwright 无头浏览器访问 Bing，获取真实搜索结果。支持 JavaScript 渲染，中文内容解析准确。

**静态爬虫（回退）**：使用 requests + BeautifulSoup，超时 15s 自动回退。

**Agent 缓存（兜底）**：预写缓存，确保离线也能演示。

### 浏览器工具检测

启动时会自动检测以下浏览器工具：
- `playwright` — Python Playwright 库
- `browser-use` — browser-use CLI
- `playwright-cli` — clawbrowser
- `stagehand` — stagehand-browser-cli

未检测到时会提示安装命令。

## 📝 输出格式

每次验证输出固定 5 个模块：

1. **结论** — 值得去 / 谨慎去 / 不建议 + 踩雷风险 + 置信度
2. **为什么这样判断** — 跨平台一致性 + 主要矛盾 + 关键证据
3. **风险标签** — 网红过热 / 游客店 / 价格不透明 / 证据不足
4. **适合谁 / 不适合谁**
5. **一句话建议**
